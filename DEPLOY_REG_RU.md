## 📤 Шаг 2: Загрузка файлов на хостинг

### Вариант A: Через FTP

1. **Получи FTP данные** из панели REG.RU:
   - FTP сервер: `ftp.ваш-домен.ru` или IP адрес
   - Логин: обычно `ваш-домен.ru` или указанный в панели
   - Пароль: из панели управления

2. **Подключись через FTP клиент** (FileZilla, WinSCP):
   ```
   Хост: ftp.ваш-домен.ru
   Пользователь: ваш-логин
   Пароль: ваш-пароль
   Порт: 21
   ```

3. **Загрузи все файлы** в папку `public_html` или `www`:
   - Все файлы из проекта
   - Структура должна сохраниться

### Вариант B: Через File Manager в панели

1. Зайди в панель управления REG.RU
2. Найди **File Manager** или **Файловый менеджер**
3. Загрузи все файлы в корневую папку сайта

## 🔑 Шаг 3: Настройка переменных окружения

### Способ 1: Через .env файл (рекомендуется)

1. Создай файл `.env` в корне сайта:
   ```env
   TELEGRAM_BOT_TOKEN=ваш_токен_бота
   TELEGRAM_CHAT_ID=ваш_chat_id
   START_DATE=1 ноября
   ```

2. **ВАЖНО:** Убедись, что `.env` не доступен извне!
   - Добавь в `.htaccess`:
   ```apache
   <FilesMatch "^\.env">
       Order allow,deny
       Deny from all
   </FilesMatch>
   ```

3. Обнови PHP файлы чтобы они читали `.env`:
   - Или используй библиотеку `vlucas/phpdotenv`
   - Или вручную добавь `parse_ini_file()` в начало PHP файлов

### Способ 2: Прямо в PHP файлах (проще, но менее безопасно)

Отредактируй `api/get-start-date.php` и `api/send-telegram.php`:
```php
$BOT_TOKEN = 'ваш_токен_здесь';
$CHAT_ID = 'ваш_chat_id_здесь';
```

**⚠️ Внимание:** Не коммить эти файлы с токенами в Git!

### Способ 3: Через переменные окружения в панели (если доступно)

Некоторые хостинги позволяют задавать переменные окружения через панель управления.

## 🔒 Шаг 6: Безопасность

### Защита .env файла

Добавь в `.htaccess`:
```apache
# Запрет доступа к .env
<FilesMatch "^\.env">
    Order allow,deny
    Deny from all
</FilesMatch>

# Запрет доступа к конфигурационным файлам
<FilesMatch "^(\.htaccess|\.git|composer\.json|package\.json)$">
    Order allow,deny
    Deny from all
</FilesMatch>
```

## 🐛 Решение проблем

### Ошибка 500 Internal Server Error

1. Проверь права доступа к файлам (должны быть 644 для файлов, 755 для папок)
2. Проверь логи ошибок в панели REG.RU
3. Убедись, что PHP версия >= 7.0

### API не работает

1. Проверь, что токены правильные в `.env` или PHP файлах
2. Проверь, что `curl` включен на сервере (обычно включен)
3. Проверь логи ошибок PHP

### Кеширование не работает

1. Убедись, что модуль `mod_expires` и `mod_headers` включены (обычно включены)
2. Проверь, что `.htaccess` загружен на сервер

## 💡 Дополнительные настройки

### Принудительный HTTPS

Раскомментируй в `.htaccess`:
```apache
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
```

### Редирект с www на без www

Раскомментируй в `.htaccess`:
```apache
RewriteEngine On
RewriteCond %{HTTP_HOST} ^www\.(.*)$ [NC]
RewriteRule ^(.*)$ https://%1/$1 [R=301,L]
```
